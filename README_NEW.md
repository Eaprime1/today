## Development Workflow

### Adding New Content
1. Identify which entity it belongs to
2. Add to appropriate entity directory
3. Update entity INVENTORY.md
4. Log change in ACCESS_LOG.md

### Creating New Entity
1. Create folder in `entities/`
2. Add standard docs (README, INVENTORY, __init__)
3. Organize content by type
4. Update root INVENTORY.md
5. Log in ACCESS_LOG.md

### Moving Concept to Naught Space
1. Ensure entity is self-contained
2. All documentation complete
3. Create corresponding structure in naught repo
4. Move entity folder
5. Log progression in ACCESS_LOG.md
